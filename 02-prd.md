# PRD: 미니 노션 (Mini Notion) — 개인 업무 관리 웹 서비스

## 1. 요약 (Summary)

이 문서는 개인 업무 관리를 위한 웹 서비스 "미니 노션"의 제품 요구사항을 정의합니다. 사용자는 구글 계정으로 로그인한 뒤, 노션과 유사한 방식으로 글(페이지)을 만들고, 편집하고, 삭제하며 자신의 업무를 관리할 수 있습니다. 유료 구독 없이 무료로 사용할 수 있고, 개인의 필요에 맞춰 기능을 자유롭게 확장할 수 있는 것이 핵심 가치입니다.

## 2. 담당자 (Contacts)

| 이름 | 역할 | 비고 |
|------|------|------|
| 제품 소유자 (본인) | PM / 개발자 / 사용자 | 요구사항 정의, 개발, 실제 사용을 모두 담당 |

> 개인 프로젝트이므로 한 사람이 기획·개발·사용을 겸합니다. 기능 우선순위 결정 권한도 제품 소유자에게 있습니다.

## 3. 배경 (Background)

- **무엇에 관한 프로젝트인가?**
  노션(Notion) 같은 업무 관리 도구는 편리하지만, 유료 요금제가 부담스럽거나 개인에게 불필요한 기능이 많습니다. 이 프로젝트는 개인 업무 관리에 꼭 필요한 핵심 기능만 담은 "미니 노션"을 직접 만드는 것입니다.

- **왜 지금인가?**
  - 상용 도구의 무료 플랜에는 사용 제한이 있고, 원하는 기능을 마음대로 추가할 수 없습니다.
  - 직접 만들면 비용이 들지 않고, 나에게 필요한 기능만 골라 넣을 수 있습니다.
  - 구글 로그인(OAuth) 등 인증 인프라가 표준화되어 있어, 개인 개발자도 안전한 로그인 기능을 쉽게 붙일 수 있게 되었습니다.

## 4. 목표 (Objective)

- **목표:** 개인 업무를 기록하고 관리할 수 있는 나만의 무료 웹 도구를 만든다.
- **왜 중요한가:**
  - 매달 나가는 구독 비용 없이 업무 관리 도구를 사용할 수 있습니다.
  - 내가 직접 만들었기 때문에, 필요할 때마다 나에게 최적화된 기능을 추가할 수 있습니다.
- **비전과의 연계:** "내 도구는 내가 만든다" — 개인 생산성 도구를 스스로 만들고 발전시키는 첫 단계입니다.

### 핵심 결과 (Key Results)

| # | 핵심 결과 | 측정 방법 |
|---|-----------|-----------|
| KR1 | 구글 로그인부터 글 작성·삭제까지 핵심 흐름이 오류 없이 동작한다 | 전체 시나리오 수동 테스트 통과 |
| KR2 | 출시 후 상용 도구 대신 이 서비스로 개인 업무를 주 3회 이상 기록한다 | 본인 사용 기록 |
| KR3 | 운영 비용 0원(무료 티어 범위 내)으로 서비스를 유지한다 | 월별 호스팅 비용 확인 |
| KR4 | 첫 버전 출시 후 3개월 내 개인 맞춤 기능을 1개 이상 추가한다 | 기능 추가 이력 |

## 5. 시장 세그먼트 (Market Segments)

- **누구를 위해 만드는가?**
  - 1차 사용자: 제품 소유자 본인 — 개인 업무를 기록하고 관리하고 싶은 사용자.
  - 잠재 사용자: 같은 문제(유료 도구 부담, 과한 기능)를 가진 개인 사용자.

- **이 세그먼트가 가진 문제(Job):**
  - "흩어져 있는 개인 업무를 한곳에 기록하고 싶다."
  - "돈을 내지 않고, 복잡하지 않은 도구를 쓰고 싶다."
  - "내 방식에 맞게 도구를 바꾸고 싶다."

- **제약 조건:**
  - 개인 프로젝트이므로 개발 리소스는 1인입니다.
  - 운영 비용은 무료(또는 무료 티어) 범위를 넘지 않아야 합니다.
  - 다중 사용자 협업 기능은 범위에서 제외합니다(개인용).

## 6. 가치 제안 (Value Propositions)

- **해결하는 고객의 일(Jobs):** 개인 업무를 글 단위로 기록·조회·정리한다.
- **고객이 얻는 것(Gains):**
  - 완전 무료로 사용하는 업무 관리 도구
  - 노션과 비슷한 익숙한 사용 방식 (`/page`로 새 글 생성 등)
  - 필요한 기능을 직접 추가할 수 있는 확장성
- **피할 수 있는 고통(Pains):**
  - 유료 구독 비용
  - 쓰지 않는 기능이 가득한 복잡한 화면
  - 외부 서비스 정책 변경(가격 인상, 기능 제한)에 휘둘리는 일
- **경쟁 대비 강점:**
  - 상용 노션 대비: 무료이고, 내 마음대로 고칠 수 있습니다.
  - 메모장/텍스트 파일 대비: 로그인 기반으로 어디서든 접근 가능하고, 글 목록·상세 구조로 정리됩니다.
  - **가치 곡선 관점:** 기능의 "폭"은 줄이고(협업·데이터베이스·템플릿 제거), "비용 없음"과 "개인 맞춤화"는 극대화합니다.

## 7. 솔루션 (Solution)

### 7.1 UX / 화면 흐름 (UX & Prototypes)

```
[로그인 페이지] --구글 로그인--> [업무 페이지 (글 목록)]
                                      │
                     ┌────────────────┼────────────────┐
                     │                │                │
              "/page" 입력       글 클릭          프로필 클릭
                     │                │                │
              [새 글 생성]      [글 상세 페이지]     [마이 페이지]
                                 - 제목 입력        - 별명 변경
                                 - 내용 입력        - 프로필 이미지
                                 - 글 삭제
```

- 로그인하지 않은 사용자는 로그인 페이지로 이동합니다.
- 로그인 후 기본 화면은 업무 페이지(글 목록)입니다.

### 7.2 핵심 기능 (Key Features)

**① 로그인 페이지**

| 기능 | 설명 |
|------|------|
| 구글 로그인 | 구글 계정(OAuth)으로 회원가입 없이 바로 로그인. 별도 비밀번호를 만들지 않아 간편하고 안전함 |

**② 마이 페이지**

| 기능 | 설명 |
|------|------|
| 별명 변경 | 서비스 안에서 사용할 별명(닉네임)을 입력·수정 |
| 프로필 이미지 | 프로필 이미지를 등록·변경 |

**③ 업무 페이지 (글 목록 & 상세)**

| 기능 | 설명 |
|------|------|
| `/page` 새 글 생성 | 입력창에 `/page`를 입력하면 새 글이 만들어짐 (노션의 슬래시 커맨드 방식) |
| 글 상세 이동 | 목록에서 글을 클릭하면 해당 글의 상세 화면으로 이동 |
| 제목·내용 편집 | 글 상세에서 제목과 내용을 입력·수정 |
| 글 삭제 | 필요 없어진 글을 삭제 |

### 7.3 기술 (Technology) — 선택 사항

- **인증:** 구글 OAuth 2.0 (예: Firebase Auth, Supabase Auth, NextAuth 등 무료 티어 활용)
- **호스팅:** 무료 티어 제공 플랫폼 (예: Vercel, Netlify 등)
- **데이터 저장:** 무료 티어 데이터베이스 (예: Supabase, Firebase 등)
- 구체적인 기술 스택은 개발 시점에 확정하되, "운영 비용 0원" 제약을 만족해야 합니다.

### 7.4 가정 (Assumptions)

검증되지 않았지만 참이라고 믿는 것들입니다. 개발 중 확인이 필요합니다.

| # | 가정 | 검증 방법 |
|---|------|-----------|
| A1 | 구글 로그인만으로 충분하다 (다른 로그인 수단 불필요) | 본인 사용 경험으로 확인 |
| A2 | 무료 티어 인프라만으로 개인 사용량을 감당할 수 있다 | 출시 후 사용량·비용 모니터링 |
| A3 | `/page` 슬래시 커맨드 방식이 버튼 방식보다 편리하다 | 실제 사용하며 비교 |
| A4 | 제목·내용의 단순 텍스트 편집만으로 초기 업무 관리가 가능하다 (서식·블록 편집 없이) | 사용하면서 불편 여부 확인 |

## 8. 출시 계획 (Release)

정확한 날짜 대신 상대적인 기간으로 계획합니다.

### 1차 버전 (MVP) — 약 2~4주

첫 버전에는 요구사항의 핵심 기능만 담습니다.

- 구글 로그인 / 로그아웃
- 업무 페이지: `/page`로 새 글 생성, 글 목록, 글 클릭 시 상세 이동
- 글 상세: 제목·내용 입력, 글 삭제
- 마이 페이지: 별명 변경, 프로필 이미지 변경

### 이후 버전 (개인 맞춤 확장) — MVP 이후 순차 진행

개인용이라는 특성을 살려, 사용하면서 필요해진 기능을 추가합니다. 후보 예시:

- 글 검색 및 정렬
- 태그 / 폴더 분류
- 할 일 체크박스, 마감일 등 업무 특화 기능
- 마크다운 또는 블록 단위 편집
- 다크 모드 등 개인 취향 기능

> 원칙: "지금 나에게 필요한가?"를 기준으로 기능을 추가하고, 협업·공유 등 다중 사용자 기능은 범위에 넣지 않습니다.
