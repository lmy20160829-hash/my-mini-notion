# 사이드바 검색 필터링 설계 — 2026-07-21

노션 Quick Find의 기초 단계. 사이드바에 이미 있는 "글 검색" 입력(`.input-sm`)에
실제 필터 로직을 붙인다. `docs/BACKLOG.md` "검색" 항목 기반 — 상단바 검색 아이콘은
건드리지 않는다(Cmd+K 팔레트는 후속 과제).

## 동작

- 입력값을 trim·소문자화해 **제목 + 본문**에 대해 부분 일치(substring)로
  사이드바 "내 글" 목록을 실시간 필터링한다.
- 빈 입력(또는 공백만) = 필터 없음(전체 표시).
- 매치 없음: `.sidebar-section__empty`로 "검색 결과가 없어요." 표시
  (기존 "아직 글이 없어요."와 구분).
- 검색 중에는 섹션 카운트 `Badge`가 **매치 개수**를 표시한다(평소엔 전체 개수).
- 메인 목록(`/`)과 글 상세는 영향받지 않는다 — 사이드바 전용.
- 대상은 활성 글만(휴지통 병렬 작업과의 접점: `posts`가 이미 활성 글만 담으므로
  추가 처리 불필요).

## 구현

- `lib/search.ts` (신규): 순수 함수
  `filterPosts(posts: Post[], query: string): Post[]`
  — trim/소문자화/빈 쿼리 전체 반환/제목·본문 substring 매치. 정렬은 입력 순서 유지.
- `components/AppShell.tsx`: `useState`로 검색어 보관, 기존 `.input-sm` input에
  value/onChange 연결, "내 글" 섹션 렌더에 `filterPosts` 적용. 접힌 레일에서는
  검색 입력 자체가 숨겨지므로(기존 동작) 추가 처리 없음.
- 외부 라이브러리·디바운스 불필요(로컬 배열 필터, 최대 수백 건).

## 테스트 (TDD)

- `lib/search.ts` 단위: 빈 쿼리/공백 쿼리 전체 반환, 제목 매치, 본문 매치,
  대소문자 무시, 매치 없음 빈 배열, 원본 불변.
- AppShell 통합: 입력 시 목록이 줄어드는지, 카운트가 매치 개수로 바뀌는지,
  매치 없음 문구, 입력 비우면 전체 복귀.
