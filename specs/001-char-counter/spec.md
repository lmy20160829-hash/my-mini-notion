# Feature Specification: 내용 글자 수 카운터 (Content Character Counter)

**Feature Branch**: `001-char-counter`

**Created**: 2026-07-10

**Status**: Draft

**Input**: User description: "내용입력칸에 내용을 입력하면 글자 수 계산하는 기능을 만들려고해. 우측 하단에 만들 예정이야"

## Clarifications

### Session 2026-07-10

- Q: 긴 글을 편집할 때 우측 하단 글자 수 표시의 배치 방식은? → A: 내용 편집 영역 우측 하단에 고정(스티키) — 글이 길어 스크롤해도 항상 같은 자리에 보임
- Q: 이모지·결합 문자(👨‍👩‍👧, 🇰🇷, 피부톤 변형 등)를 몇 글자로 셀까? → A: 사용자가 하나로 인식하는 글자(grapheme cluster) 단위로 각 1글자

## User Scenarios & Testing *(mandatory)*

이 기능은 글 상세 화면의 **내용 입력칸**에 대해, 지금까지 입력된 **글자 수**를 화면 **우측 하단**에 실시간으로 보여준다. 사용자는 자신이 얼마나 썼는지 별도의 조작 없이 즉시 확인할 수 있다.

### User Story 1 - 입력하면서 실시간으로 글자 수 확인 (Priority: P1)

사용자가 글 상세 화면의 내용 입력칸에 글을 작성한다. 한 글자를 입력할 때마다 화면 우측 하단의 글자 수 표시가 즉시 갱신되어, 지금까지 몇 글자를 썼는지 항상 알 수 있다.

**Why this priority**: 이 기능의 핵심 가치다. 실시간 카운트가 없으면 기능 자체가 성립하지 않는다. 이 스토리 하나만 구현해도 "쓰는 만큼 글자 수를 본다"는 사용자 가치를 온전히 전달하는 MVP가 된다.

**Independent Test**: 빈 내용 입력칸에 글자를 하나씩 입력하면서, 우측 하단 표시가 입력한 글자 수와 정확히 일치하며 갱신되는지 확인한다. 이것만으로 완결된 가치를 검증할 수 있다.

**Acceptance Scenarios**:

1. **Given** 내용 입력칸이 비어 있고 글 상세 화면이 열려 있을 때, **When** 사용자가 "안녕"을 입력하면, **Then** 우측 하단에 "2"(글자 수)가 표시된다.
2. **Given** 내용 입력칸에 이미 글자가 있을 때, **When** 사용자가 글자를 추가로 입력하면, **Then** 표시된 글자 수가 추가된 만큼 즉시 증가한다.
3. **Given** 사용자가 입력 중일 때, **When** 각 키 입력이 반영되면, **Then** 별도 버튼 클릭이나 저장 없이 표시 값이 자동으로 갱신된다.

---

### User Story 2 - 기존 글을 열면 현재 글자 수가 바로 보임 (Priority: P2)

사용자가 이미 내용이 저장된 글을 연다. 아무것도 입력하지 않아도, 화면 우측 하단에 그 글의 현재 글자 수가 즉시 표시된다.

**Why this priority**: 기존 글을 이어서 작성하는 경우가 흔하므로, 처음 입력할 때만이 아니라 열자마자 정확한 값을 보여줘야 신뢰할 수 있다. P1이 "입력 중 갱신"이라면 이 스토리는 "초기 상태의 정확성"을 보장한다.

**Independent Test**: 내용이 채워진 글을 연 직후, 입력을 전혀 하지 않은 상태에서 우측 하단 글자 수가 실제 저장된 내용의 글자 수와 일치하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 내용이 120글자 저장된 글이 있을 때, **When** 사용자가 그 글의 상세 화면을 열면, **Then** 우측 하단에 "120"이 표시된다.
2. **Given** 한 글의 상세 화면을 보고 있을 때, **When** 사용자가 다른 글로 이동하면, **Then** 글자 수 표시가 새로 연 글의 내용 기준으로 갱신된다.

---

### User Story 3 - 삭제·붙여넣기·빈 내용까지 정확히 반영 (Priority: P3)

사용자가 내용을 지우거나, 여러 줄을 한꺼번에 붙여넣거나, 전부 삭제한다. 어떤 방식으로 내용이 바뀌든 우측 하단 글자 수는 항상 현재 내용과 일치한다.

**Why this priority**: 입력뿐 아니라 편집(삭제·붙여넣기·잘라내기)에서도 값이 어긋나지 않아야 카운터를 믿을 수 있다. 핵심 흐름은 아니지만 정확성에 대한 신뢰를 완성한다.

**Independent Test**: 내용을 붙여넣기·부분 삭제·전체 삭제로 바꿔 가며, 매 순간 표시 값이 실제 글자 수(비었을 때 0 포함)와 일치하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 내용 입력칸에 50글자가 있을 때, **When** 사용자가 30글자를 삭제하면, **Then** 표시 값이 "20"으로 갱신된다.
2. **Given** 내용 입력칸이 비어 있을 때, **When** 사용자가 200글자 텍스트를 붙여넣으면, **Then** 표시 값이 "200"으로 갱신된다.
3. **Given** 내용 입력칸에 글자가 있을 때, **When** 사용자가 전체를 삭제해 비우면, **Then** 표시 값이 "0"이 된다.

---

### Edge Cases

- **빈 내용**: 내용이 전혀 없으면 글자 수는 0으로 표시된다(표시가 사라지지 않고 0을 보여준다).
- **공백·줄바꿈만 있는 내용**: 스페이스, 탭, 줄바꿈도 글자로 세어 포함한다(공백 포함 기준).
- **매우 긴 내용**: 수천 글자 이상이어도 표시 값이 정확하고, 입력 반응이 눈에 띄게 느려지지 않는다.
- **한글·이모지 등**: 사용자가 하나로 인식하는 글자(grapheme cluster)를 1글자로 센다. 한글 음절은 각각 1글자, 결합 이모지(👨‍👩‍👧)·국기(🇰🇷)·피부톤 변형 이모지도 각각 1글자로 계산한다.
- **제목 입력칸**: 글자 수는 내용 입력칸만 대상으로 하며, 제목 입력칸의 글자는 세지 않는다.
- **좁은 화면/스크롤**: 우측 하단 표시가 내용 텍스트를 가리거나 편집을 방해하지 않는다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 글 상세 화면의 **내용 입력칸**에 대한 글자 수를 화면에 표시해야 한다(MUST).
- **FR-002**: 글자 수 표시는 사용자가 입력하는 동안 **실시간으로** 갱신되어야 한다(별도의 버튼·저장·새로고침 불필요).
- **FR-003**: 글 상세 화면을 열면, 입력을 하지 않아도 현재 저장된 내용의 글자 수가 **즉시** 표시되어야 한다.
- **FR-004**: 글자 수는 입력뿐 아니라 삭제·붙여넣기·잘라내기 등 모든 내용 변경을 정확히 반영해야 한다.
- **FR-005**: 내용이 비어 있으면 글자 수는 **0**으로 표시되어야 한다.
- **FR-006**: 글자 수 표시는 내용 편집 영역의 **우측 하단에 고정(스티키)**되어, 내용이 길어 스크롤하는 동안에도 항상 같은 위치에 보여야 한다.
- **FR-007**: 글자 수 표시는 사용자의 내용 읽기·편집을 가리거나 방해하지 않아야 한다.
- **FR-008**: 글자 수는 **내용 입력칸만** 대상으로 하며, 제목 입력칸은 포함하지 않는다.
- **FR-009**: 글자 수는 공백과 줄바꿈을 포함한 **모든 문자**를 세되, 사용자가 하나로 인식하는 글자(**grapheme cluster**) 단위로 계산한다. 결합 이모지(👨‍👩‍👧)·국기(🇰🇷)·피부톤 변형 이모지·한글 음절은 각각 1글자로 센다.
- **FR-010**: 글자 수 표시는 그 값이 "글자 수"임을 사용자가 알 수 있도록 명확히 나타내야 한다(숫자와 함께 적절한 라벨/단위 사용).
- **FR-011**: 다른 글로 이동하면 글자 수 표시는 새로 연 글의 내용 기준으로 갱신되어야 한다.

### Key Entities *(include if feature involves data)*

- **글 내용(Note Content)**: 글 상세 화면의 내용 입력칸에 담긴 텍스트. 이미 존재하는 데이터이며, 이 기능은 이를 새로 저장하지 않는다.
- **글자 수(Character Count)**: 글 내용에서 파생되는 값(공백·줄바꿈 포함 총 문자 수). 별도로 저장하지 않고 현재 내용으로부터 실시간 계산되는 표시 전용 값이다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 내용 입력칸에 글자를 입력·삭제할 때, 표시되는 글자 수가 즉시(사람이 지연을 느끼지 못하는 수준으로) 갱신된다.
- **SC-002**: 임의의 내용(빈 값, 공백만, 긴 텍스트, 붙여넣기 포함)에 대해 표시된 글자 수가 실제 글자 수와 **100% 일치**한다(값 어긋남 0건).
- **SC-003**: 사용자는 내용을 편집하는 동안 스크롤 없이 우측 하단에서 글자 수를 확인할 수 있다.
- **SC-004**: 기능을 처음 접한 사용자의 90% 이상이 별도 안내 없이 "이 숫자가 글자 수"임을 5초 이내에 인지한다.

## Assumptions

- "내용입력칸"은 글 상세 화면의 **내용(본문) 입력칸**을 가리키며, 제목 입력칸은 포함하지 않는다.
- "글자 수"는 공백과 줄바꿈을 포함한 **총 문자 수**를 의미한다. 단어 수 세기, 공백 제외 세기는 v1 범위 밖이다.
- 이 기능은 **표시 전용**이다. 최대 글자 수 제한이나 입력 차단은 v1 범위 밖이다.
- "우측 하단"은 내용 편집 영역의 우측 하단에 **고정(스티키)**되는 위치를 의미하며, 스크롤과 무관하게 항상 보이고 편집을 방해하지 않는다. (정확한 좌표·스타일은 `DESIGN.md`에 따라 계획 단계에서 확정)
- 내용 입력칸과 자동 저장 기능은 이미 존재하며, 이 기능은 그 위에 **글자 수 표시만** 추가한다.
- 데스크톱 웹을 우선 대상으로 한다(기존 앱과 동일).
- 파생 값인 글자 수는 별도로 저장하지 않는다.
